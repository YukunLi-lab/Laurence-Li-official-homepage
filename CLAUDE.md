# CLAUDE.md - 个人学术网站工作流

## 项目概述

使用 Quarto 构建的个人学术网站，用于研究生申请展示。

- **框架**: Quarto
- **风格**: 创意展示型 - 现代简约，突出个人特色
- **主色调**: 深蓝灰 (#2C3E50)，柔和绿 (#7FB069) 作为点缀色
- **部署**: GitHub Pages

## 内容板块（8个）

1. 关于我 (Hero) - 姓名、头像、自我介绍
2. 教育背景 - 学校、GPA、核心课程
3. 科研经历 - 项目、角色、方法、成果
4. 项目展示 - 课程项目、技术栈
5. 成果发表 - 论文、会议展示
6. 技能 - 编程语言、研究工具
7. 荣誉奖项 - 奖学金、竞赛
8. 联系方式 - 邮箱、GitHub、LinkedIn、CV下载

## 工作流规则

### 内容管理

- 占位符格式：`[您的数据]`、`[占位符]` - 后续需替换为真实内容
- CV 放在 `assets/cv.pdf`
- 页面标题格式："姓名 | 学术主页"
- 更新内容时同步更新 `lastupdated` 日期

### 学术成果更新

- 新增发表论文需包含完整引用格式
- 科研经历需包含具体贡献和量化指标（如准确率、提升幅度等）
- 添加会议展示和海报展示
- 确保所有外部链接（论文、项目）有效

### 质量标准

- 使用专业、学术的语气
- 使用主动语态和具体成果描述
- 尽可能量化成果（准确率 %、提升幅度等）
- 保持格式一致性
- 链接颜色使用柔和蓝 (#2980B9)

### 文件组织

```
images/profile/      - 头像 (500x500px, 命名 headshot.png)
images/projects/    - 项目截图
assets/cv.pdf       - 最新简历 PDF（建议小于 2MB）
```

### 部署命令

```bash
# 本地预览
quarto preview

# 构建部署版本
quarto render

# 发布到 GitHub Pages
quarto publish gh-pages
```

### 发布前检查清单

- [ ] 所有 `[占位符]` 文本替换为真实内容
- [ ] 头像已上传 (500x500px, 命名 `headshot.png`)
- [ ] CV PDF 已更新并同步
- [ ] 所有外部链接验证有效
- [ ] GitHub 仓库已设置为 public
- [ ] GitHub Pages 已启用并配置
- [ ] 自定义域名已配置（如适用）
- [ ] 移动端响应式测试通过
- [ ] 浏览器控制台无错误

### Git 工作流

1. 创建新内容分支：`git checkout -b update/ publications`
2. 修改并提交：`git commit -m "Add publication: 论文标题"`
3. 推送并创建 PR：`git push origin update/publications`
4. 审核后合并到 main
5. GitHub Actions 自动部署到 GitHub Pages

### 响应指南

- 保持专业、学术的语气
- 使用主动语态和具体成果
- 量化成果（准确率 %、改进效果等）
- 保持格式一致性
- 主色调作为强调色使用（深蓝灰 #2C3E50）
